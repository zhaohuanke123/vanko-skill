# 费曼深度验证机制

> **本文档说明如何在对话中进行深度验证，记录每一轮追问过程。**
>
> **AI读取时机：** 用户说"准备好了"开始检查站验证时

---

## 课程文件中的检查站格式

课程文件只写"检查站主题"，不写具体问题：

```markdown
## 检查站

> [!info] 本课核心概念
> - 概念1：简要描述
> - 概念2：简要描述
> - 概念3：简要描述

**AI将根据以上主题，在对话中逐一提问验证。**

---
请告诉我：`准备好了` - 开始检查站验证
- `深入探讨 [概念]` - 对某个概念进行分支探索
- `纠错 [内容]` - 对课程内容提出质疑和纠错
```

---

## 核心原则

**问题不在课程文件中预设，AI 在对话中动态生成。**

这样做的优势：
- AI 必须追问，无法跳过验证环节
- 用户无法"背诵答案"
- 追问自然发生，对话流畅
- 辩论过程完整记录，学习轨迹可见

---

## 验证流程

```
用户说"准备好了"
    │
    ├── AI读取课程文件的"检查站主题"
    │
    ├── 选择第一个主题
    │
    ├── 生成第一个问题
    │   └── 问题基于主题 + 课程上下文 + 用户之前的回答
    │
    ├── 用户回答
    │
    ├── AI评估回答，选择策略：
    │   │
    │   ├── 策略A：追问原因（回答正确但可能表面）
    │   │   "为什么是这个而不是那个？底层原因是什么？"
    │   │
    │   ├── 策略B：要具体例子（回答模糊）
    │   │   "能具体说说吗？举个例子？"
    │   │
    │   ├── 策略C：问推理过程（回答错误）
    │   │   "你是怎么得出这个结论的？"
    │   │
    │   ├── 策略D：换情境验证（怀疑是背诵）
    │   │   "如果换成X情况，结果会怎样？"
    │   │
    │   └── 策略E：让用户教AI（终极验证）
    │       "假设我是新手，你会怎么向我解释？"
    │
    ├── 用户回应追问
    │
    ├── 判断掌握程度：
    │   ├── L1 复述 → 继续追问
    │   ├── L2 解释 → 继续追问或换情境验证
    │   ├── L3 应用 → 该主题通过 ✅
    │   └── 用户说"理解了" → 该主题通过 ✅
    │
    ├── 记录本轮对话到课程文件
    │
    ├── 进入下一个主题（如有）
    │
    └── 所有主题通过 → 检查站完成
```

---

## 验证策略详解

### 策略A：追问原因

**触发条件**：用户回答正确，但可能只是记忆或表面理解。

**话术**：
```
"你说得对。不过我想多问一句：为什么X会导致Y而不是Z？底层的原因是什么？"
```

**示例**：
```
用户：闭包就是函数能访问定义时的变量。
AI：正确。那我问你：为什么是"定义时"而不是"调用时"？这背后是什么机制？
```

### 策略B：要具体例子

**触发条件**：用户回答模糊、抽象、缺少具体性。

**话术**：
```
"能具体说说吗？给我一个例子？"
"你提到X，能举个具体的场景吗？"
```

**示例**：
```
用户：周期股就是随经济周期波动的股票。
AI：能举个具体的周期股例子吗？它的盈利是怎么随周期变化的？
```

### 策略C：问推理过程

**触发条件**：用户回答错误，需要找到理解断层。

**话术**：
```
"你是怎么得出这个结论的？能说说你的推理过程吗？"
```

**目的**：让用户自己发现推理漏洞，而非直接纠正。

**示例**：
```
用户：PE越低越好，5倍PE肯定便宜。
AI：你是怎么得出"PE越低越好"这个结论的？能说说你的推理吗？
用户：PE低=便宜=划算...
AI：那我问你：如果一家公司PE只有5倍，但明年可能亏损，还便宜吗？
```

### 策略D：换情境验证

**触发条件**：怀疑用户是背诵而非真理解。

**话术**：
```
"如果换成X情况，结果会怎样？为什么？"
```

**示例**：
```
用户：周期股PE低要卖出，PE高要买入。
AI：那如果一家公司PE高是因为刚从亏损转为微利呢？还是买入机会吗？
```

### 策略E：让用户教AI

**触发条件**：终极验证，检验用户能否用自己的话解释清楚。

**话术**：
```
"很好！现在假设我是个完全不懂这个概念的新手，你会怎么向我解释？用最简单的话。"
```

**这是 L4 迁移级别的验证。**

---

## 掌握程度判断

| 等级 | 判断标准 | AI行动 |
|-----|---------|--------|
| L1 复述 | 能说出关键词 | 继续追问，要求解释 |
| L2 解释 | 能用自己的话解释 | 换情境验证 |
| L3 应用 | 能处理变体问题 | ✅ 该主题通过 |
| L4 迁移 | 能向新手解释清楚 | ✅ 该主题通过（优秀） |
| L5 创造 | 能主动关联其他知识 | ✅ 该主题通过（卓越） |

**最低通过标准：L3 应用级别**

**用户主动确认也算通过**：用户说"我理解了"、"明白了"、"懂了"

## 对话记录格式

每轮追问结束后，记录到课程文件末尾：

```markdown
---
## 检查站对话记录

### 主题1：[主题名称]

**AI提问**：[第一个问题]
**用户回答**：[用户回答]
**AI追问**：[追问内容]
**用户回答**：[用户回答]
...（继续记录直到通过）
**掌握程度**：L3 应用级别 ✅
**耗时**：N轮追问

### 主题2：[主题名称]

**AI提问**：[第一个问题]
**用户回答**：[用户回答]
**掌握程度**：L3 应用级别 ✅
**耗时**：1轮追问

---
**检查站完成时间**：YYYY-MM-DD HH:MM
**总耗时**：N轮追问
```

## 实战示例

### 示例：PE估值课程

**课程检查站主题**：
- PE估值的陷阱
- 周期股的特殊性

**对话过程**：

```
AI：准备好了？我们开始验证。第一个主题：PE估值的陷阱。
    某钢铁股PE只有5倍，能说它便宜吗？

用户：不知道。

AI：这个名字本身有提示——"钢铁"。你觉得钢铁行业有什么特点？

用户：和经济周期有关？

AI：对！那经济周期对盈利有什么影响？

用户：经济好盈利高，经济差盈利低。

AI：那盈利高的时候PE会怎样？

用户：盈利高PE就低...哦！所以PE低可能是盈利最高的时候！

AI：很好！那盈利最高意味着什么？

用户：意味着接下来可能要走下坡路了...所以PE低反而是卖点！

AI：完美！你已经理解了周期股PE陷阱的本质。
    掌握程度：L3 应用级别 ✅
    
    下一个主题：周期股的特殊性。
    为什么周期股在盈利最差的时候，股价反而可能上涨？

用户：因为市场预期接下来会复苏？

AI：能说说你的推理过程吗？

用户：周期股盈利随经济周期波动。盈利最差时可能是周期谷底，
    市场知道接下来大概率复苏，所以提前买入，股价上涨。

AI：很好！你已经能用推理链条解释这个现象。
    掌握程度：L3 应用级别 ✅
    
    检查站完成！
```

**记录到课程文件**：

## AI 执行检查清单

每次用户回答后：

- [ ] 不直接判断对错，先选择追问策略
- [ ] 使用正确的话术（原因/例子/推理过程/换情境/教AI）
- [ ] 判断掌握程度（L1-L5）
- [ ] 记录本轮对话到课程文件
- [ ] 达到L3或用户说"理解了"才算通过
- [ ] 通过后进入下一个主题或完成检查站

# 分支探索机制

> **本文档说明如何触发分支、探索分支、回归主线，以及跨对话恢复分支状态。**
>
> **AI读取时机：** 用户说"深入探讨X"、"回到主线"、"继续学习"时
>
> **配置引用：** 分支深度限制见 `config.json` → `teaching.maxBranchDepth`

---

## 触发条件

当用户说：
- "深入探讨X"
- "我对X还有疑问"
- "能详细讲讲X吗"
- "分支学习X"

---

## 分支流程

```
主线课程 (02_核心原理.md)
    │
    ├── 用户想深入了解"闭包"
    │
    ├── 创建分支文件 (02a_闭包深入.md)
    │   ├── 文件头部标注：分支来源: [[02_核心原理]]
    │   ├── 内容聚焦闭包专题
    │   └── 独立的检查站问题
    │
    ├── 用户继续探索或回答问题
    │
    └── 用户说"回到主线"
        │
        ├── 总结分支收获（3-5条要点）
        ├── 更新知识图谱.md
        ├── 更新进度.md
        └── 回到主线课程，继续下一课
```

## 回归主线流程

当用户说"回到主线"或分支学习完成时：

1. **生成分支收获摘要**（自动或用户确认）
   ```markdown
   ## 分支收获：闭包深入
   - 闭包 = 函数 + 词法环境
   - 闭包"记住"变量的原理是作用域链
   - 常见应用：工厂函数、回调、装饰器
   ```

2. **更新知识图谱.md**
   - 标记分支状态为"已完成"
   - 记录关键收获

3. **更新主线课程的链接**
   - 在主线课程中添加反向链接：`延伸学习：[[02a_闭包深入]]`

4. **更新进度.md**
   - 当前状态：学习中
   - 活跃分支：无

## 跨对话分支恢复

### 进度文件关键字段

```markdown
## 恢复点（新对话必读）

> [!important] 快速恢复
> - **当前状态**：分支探索中 ← AI据此判断是否在分支
> - **当前位置**：[[02_核心原理]] ← 主线位置
> - **活跃分支**：[[02a_闭包深入]] ← 当前分支
> - **下一步**：完成闭包深入分支的检查站问题
```

```
正在学习闭包概念。要继续吗？"
```

**场景B：分支探索中恢复**

```
- 活跃分支：[[02a_闭包深入]]
```

## 触发与恢复完整流程

### 对话内触发分支

```
用户: 深入探讨闭包

AI执行：
1. 读取当前课程（02_核心原理.md）
2. 提取上下文摘要
3. 创建分支文件（02a_闭包深入.md）
4. 更新进度.md：
   - 当前状态：分支探索中
   - 活跃分支：[[02a_闭包深入]]
5. 展示分支课程内容
```

### 新对话恢复分支

```
用户: 继续学习

AI执行：
1. 扫描学习目录
2. 读取进度.md
3. 发现"当前状态：分支探索中"
4. 读取活跃分支文件（02a_闭包深入.md）
5. 展示恢复选项，让用户选择继续分支或回到主线
```

## 分支深度建议

```
主线 (02_核心原理)
    │
    ├── 分支1 (02a_闭包深入)      ← 建议在单次对话内完成
    │   │
    │   └── 子分支 (02a1_词法作用域) ← 考虑开新对话或快速回归
    │
    └── 分支2 (02b_装饰器)
```

**建议：**
- 分支深度不超过2层
- 深分支完成后立即回归主线
- 复杂分支可以单独开对话探索

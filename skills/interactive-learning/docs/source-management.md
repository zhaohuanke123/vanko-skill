# 来源引用机制

> **本文档说明如何基于用户提供的材料生成课程，减少AI幻觉。**
>
> **AI读取时机：** 用户发起学习主题时

---

## 核心问题

AI生成课程时可能"编"细节——形式正确但具体内容有误。这是AI的概率预测本质决定的。

**解决方案**：让AI基于用户提供的原始材料生成课程，并提供准确的引用。

---

## 来源类型

| 类型 | 提供方式 | 可信度 | 图标 |
|-----|---------|-------|------|
| 本地文件 | 文件路径（PDF、MD、TXT） | 高 | 📚 |
| Obsidian笔记 | `[[笔记名]]` 链接 | 高 | 📚 |
| 书籍引用 | 书名 + 章节 + 作者 | 高 | 📖 |
| 权威网站 | 官方文档、维基百科等 | 高 | 🌐 权威 |
| 一般网站 | 博客、论坛等 | 中 | 🌐 一般 |
| 待验证来源 | 来源不明 | 低 | 🌐 待验证 |

---

## 用户输入方式

### 方式1：主动提供材料

```
我想学习投资理财，参考资料：
- 文件：./books/聪明的投资者.pdf
- 网站：https://www.investopedia.com/terms/p/price-earningsratio.asp
- Obsidian笔记：[[投资笔记/PE估值]]
- 书籍：《巴菲特致股东的信》第3章
```

### 方式2：仅提供主题

```
我想学习投资理财入门
```

AI会主动搜索网络来源，并标注可信度。

### 方式3：混合模式

```
我想学习Python闭包，参考资料：[[编程笔记/函数基础]]
```

AI会使用用户提供的笔记，并补充搜索其他来源。

---

## AI处理流程

```
用户发起学习主题
    │
    ├── 检查是否提供来源材料
    │   │
    │   ├── 有来源材料
    │   │   ├── 本地文件 → 读取内容
    │   │   ├── Obsidian笔记 → 读取笔记 + 准备添加块ID
    │   │   ├── 网站链接 → WebFetch获取内容
    │   │   └── 书籍引用 → 使用AI知识 + 标注来源
    │   │
    │   └── 无来源材料
    │       └── 检查 config.json → sources.defaultSearchEngine
    │           ├── "web" → AI主动搜索网络
    │           └── "none" → 仅使用AI知识（标注"待验证"）
    │
    ├── 提取与主题相关的关键内容
    │
    ├── 生成课程内容
    │   └── 每个关键知识点后标注来源
    │
    └── 课程末尾列出参考文献
```

## 引用格式

### 在正文中引用

```markdown
PE估值是用股价除以每股收益得出的指标[^1]。

[^1]: 📚 [[投资笔记/PE估值#^pe-def]]
```

### 参考文献列表

```markdown
## 参考文献

> [!quote] 本课参考资料
> 1. 📚 [[投资笔记/PE估值#^pe-def]] - PE定义 - **本地笔记**
> 2. 🌐 [Investopedia - PE Ratio](https://www.investopedia.com/terms/p/price-earningsratio.asp) - PE详解 - **权威网站**
> 3. 📖 《聪明的投资者》第8章 - Benjamin Graham - **经典书籍**
```

## Obsidian 块ID自动添加

当AI需要引用Obsidian笔记中的特定段落时，自动在原文段落末尾添加 `^block-id`。

### 示例

**原文（投资笔记/PE估值.md）：**
```markdown
## 什么是PE

PE估值是用股价除以每股收益得出的指标
```

**AI操作：**

1. 读取笔记内容
2. 找到要引用的段落
3. 自动添加块ID：
   ```markdown
   ## 什么是PE

   PE估值是用股价除以每股收益得出的指标 ^pe-def
   ```
4. 在课程中引用：
   ```markdown
   PE估值是衡量股票价格是否合理的重要指标[^1]。

   [^1]: 📚 [[投资笔记/PE估值#^pe-def]]
   ```

### 块ID命名规则

- 使用有意义的前缀：`pe-def`、`pe-calc`、`pe-trap`
- 小写字母 + 连字符
- 简短但能表达含义

## 可信度评估标准

### 高可信度

- 📚 **本地笔记**：用户自己的材料，信任度最高
- 📖 **经典书籍**：经时间验证的出版物
- 🌐 **权威网站**：
  - 官方文档（如 MDN、Python.org）
  - 维基百科
  - 知名教育网站（如 Investopedia、Khan Academy）
  - 学术机构

### 中可信度

- 🌐 **一般网站**：
  - 个人博客
  - 论坛帖子
  - 知乎回答
  - Medium文章

### 低可信度

- 🌐 **待验证**：
  - 社交媒体
  - 匿名来源
  - 缺乏引用的文章

## AI主动搜索策略

当用户未提供来源时，AI按以下顺序搜索：

1. **官方文档优先**：搜索 `[主题] official documentation`
2. **权威网站次之**：搜索 `[主题] site:wikipedia.org` 或 `site:investopedia.com`
3. **一般网站补充**：搜索 `[主题] 教程` 或 `[主题] 入门`

搜索后标注可信度，并在参考文献中列出。

---

## 幻觉减少策略

1. **基于材料生成**：优先使用用户提供的材料
2. **标注来源**：每个知识点都有出处
3. **区分可信度**：让用户知道哪些来源更可靠
4. **鼓励纠错**：用户可通过 `纠错 [内容]` 指出错误
5. **保留原文链接**：用户可跳转到原始材料验证

## 配置说明

```json
{
  "sources": {
    "obsidianVault": "./笔记",     // Obsidian库路径，null表示不启用
    "defaultSearchEngine": "web"   // AI主动搜索行为：web/none
  }
}
```

| 字段 | 说明 | 默认值 |
|-----|------|-------|
| `obsidianVault` | Obsidian 库路径，用于自动添加块ID | `null` |
| `defaultSearchEngine` | AI主动搜索行为 | `web` |

## 实战示例

### 示例1：用户提供Obsidian笔记

**用户输入：**
```
我想学习Python闭包，参考资料：[[编程笔记/函数基础]]
```

**AI操作：**
1. 读取 `编程笔记/函数基础.md`
2. 提取与闭包相关的内容
3. 在原文添加块ID：`闭包是函数能够访问定义时的作用域中的变量 ^closure-def`
4. 生成课程，引用：`[[编程笔记/函数基础#^closure-def]]`

### 示例2：AI主动搜索

**用户输入：**
```
我想学习投资理财入门
```

**AI操作：**
1. 搜索 `投资理财入门 官方指南`
2. 搜索 `investing for beginners investopedia`
3. 综合多个来源生成课程
4. 标注可信度，列出参考文献

### 示例3：混合模式

**用户输入：**
```
我想学习股票估值，参考资料：《聪明的投资者》
```

**AI操作：**
1. 使用《聪明的投资者》内容（AI知识 + 书籍引用）
2. 补充搜索 `stock valuation investopedia`
3. 生成课程，标注两类来源
